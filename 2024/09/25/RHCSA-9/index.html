3. Control Services and Daemon

Service

  • 부팅 때 자동으로 켜지고 계속 백그라운드에서 실행되는 프로세스
  • 메모리에 상주해 있다가 사용자 요청이 오면 응답하는 프로세스
  • 서비스는 눈에 보이지 않지만 현재 시스템에서 동작 중인 프로세스이므로 백그라운드 프로세스의 일종이라고 할 수 있다.

Deamon

  • 부팅 때 자동으로 켜지고 계속 백그라운드에서 실행되는 프로세스
  • 주로 시스템의 특정 작업이나 서비스를 관리한다.
  • 예: cron (스케줄 관리), sshd (SSH 서비스), httpd (웹 서버), systemd

3-1. systemd

  • 서비스 매니저 프로그램으로 서비스를 작동시키거나 관리한다
  • OS 부팅 시 운영체제 중 최초로 시작되는 프로세스 1번 PID
  • 리눅스의 정상적인 부팅을 위해 초기화 해주는 역할 + 시스템을 총 관리하는 역할

command

3-2. sshd

  • secure shell
  • 여기서 shell 은 말그대로 껍질을 의미하며 리눅스 시스템과의 인터페이스를 제공한다.
  • shell 은 사용자의 command를 커널에게 번역해서 하드웨어를 관리할 수 있도록한다.
  • SSH는 항상 암호화 되어있어 보안성이 높지만 /etc/ssh/sshd_config 설정을 통해 보안성을 더 높일 수 있다.

4-1. nmcli

  • 리눅스는 시스템에서 네트워크 설정과 관리리를 쉽게 할 수 있도록 NetworkManager를 제공한다. nmcli은 커맨드 라인에서 이를 사용할 수 있도록 한다.

OBJECT: 관리할 네트워크 객체 (예: connection, device, general 등)
COMMAND: 실행할 작업 (예: show, up, down, add, delete 등)
OPTIONS: 선택적으로 사용할 수 있는 플래그나 매개변수

5-1. tar

  • 여러 개의 파일을 하나의 파일로 묶거나 풀 때 사용하는 명령어로 “테이프 아카이버(Tape ARchiver)”의 앞 글자들을 조합하여 만들었다.

command

5-2. ftp and scp

  • FTP (File Transfer Protocol) : 파일을 전송하는 통식 규약으로 기본 폴트는 21이지만 설정을 통해 변경해서 사용 할 수 있다. FTP는 암호화가 되지 않아서 tcp dump를 뜨면 내용이 다 보이기 때문에 실무에서는 사용하지 않는다.

  • SCP (Secure Copy Protocol) : SSH 프로토콜 기반의 파일 전송(복사) 수단이다. 네트워크 연결환경에서 ssh와 동일한 22번 포트와 identy file을 사용해서 파일을 전송하기 때문에 보안적으로 안정된 프로토콜이다.

command


6. System Updates and Repos

리눅스 패키지(Linux Package)란 리눅스 시스템에서 소프트웨어를 실행하는데 필요한 파일들(실행 파일, 설정 파일, 라이브러리 등)이 담겨 있는 설치 파일 묶음이다. 리눅스 패키지는 말그대로 소스코드가 들어있는 소스 패키지(Source Package) 와 컴파일된 바이너리 파일이 들어 있는 바이너리 패키지(Binary Package) 가 있다. 소스 패키지는 컴파일 과정을 거쳐 바이너리 파일로 만든후 사용이 가능하기 때문에 리눅스의 기본 설치 패키지들은 소스 패키지에 비해 설치 시간도 짧고 오류가 발생할 가능성도 적은 바이너리 패키지로 되어있다.

이러한 패키지를 관리하기 위해서는 패키지 관리 도구를 사용하는데 일반적으로 두 유형의 패키지 관리 도구가 사용된다.

  • yum (CentOS) : 인터넷 연결이 필요하다
  • apt-get (other Linux)
  • rpm (Redhat Package Manager) : 인터넷이 없어도 /var/lib/rpm 에 위치하는 데이터베이스를 기반으로 페키지를 설치 및 관리 할 수 있다.

7. Podman Container

Podman (Pod Manager tool) 는
컨테이너화된 애플리케이션을 관리하기 위한 오픈 소스 도구로, Docker와 유사한 기능을 제공하는 container engine이다.

  • 주로 리눅스 환경에서 사용
  • 컨테이너, 이미지, 볼륨, 네트워크를 관리할 수 있다.
  • Daemonless : Podman의 가장 큰 특징중 하나는 데몬이 필요없다는 것이다.
  • Docker는 중앙 데몬 프로세스를 통해 컨테이너를 관리하는 반면, Podman은 개별 컨테이너를 독립적으로 실행하고 관리할 수 있어 높은 보안성과 유연성을 제공한다. ( 컨테이너를 사용자의 권한으로 실행할 수 있다.)

리눅스 * 그냥 재미로

리눅스는 핀란드 출신의 공대생 리누스 토르발스(Linus Torvalds)가 처음 개발했다. 그는 MINIX라는 유닉스 계열의 운영체제을 접하면서 불편했던 점이 생겨 그냥 “재미로” 리눅스를 만들어 버렸다… 그렇게 리누스 토르발스와 GNU 프로젝트의 합작으로 리눅스는 세상에 태어나게 된다. 그리고 오픈소스 즉 무료였던 리눅스는 세간의 주목을 받고 여러 배포판이 생겨나게 되었다. 이제 서버, 모바일 우주정거장에도 리눅스 운영체제를 기반으로 시스템이 존재한다고 한다.

추가로 리눅스의 마스코드인 귀여운 펭귄 턱스는 리루스 토르발스가 작은 푸른펭귄을 좋아해서 “행복하고 만족스러운” 모습의 펭귄 턱스가 리눅스의 마스코트다 되었다고 한다ㅎㅎ ( 귀여워… )
