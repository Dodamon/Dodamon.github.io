Linux:Booting

2024-11-02

리눅스 운영체제를 공부하면서 새로 알게된 점이나 신기한 내용들을 정리하려한다. 그래서 이번 주제는 바로 “부팅(Booting)” 이다.

Booting 이란

Booting은 디스크에 저장되어 있는 운영체제를 메모리에 올리는 것이다.

  • 메모리(RAM, Random Access Memory) : 주기억장치로, 전원이 꺼지면 데이터가 사라진다.
  • 디스크(HDD/SSD) : 보조 기억, 장치로, 전원이 꺼져도 영구적으로 데이터를 저장한다.

처음으로 컴퓨터를 켜면 부팅화면이 보인다 이때 컴퓨터는 전원이 꺼지면서 내려간 운영체제를 다시 실행하기 위헤 디스크에서 메모리로 로드하는 작업을 한다.
이것을 우리는 부팅이라고 한다. 각각의 순서를 자세히 보면

각각의 순서는 크게 다음과 같다.

  1. 전원을 켠다. ( 전기가 들어온다 )
  2. 하드웨어가 정상적으로 작동하는지 확인한다.
  3. 운영체제를 메모리에 로드한다.

2번 부터 더 자세하게 알아보자

2-1. BIOS(Basic Input Output System) / UEFI (Unified Extensible Firmware Interface) 펌웨어 실행

POST(Power-On Self-Test)

컴퓨터에 전원이 들어오게 되면 CPU가 마더보드에 있는 ROM(비휘발성 메모리)에 있는 BIOS/UEFI라는 펌웨어를 메모리에 올려 실행하게 된다. 그중 가장 먼저 POST(Power-On Self-Test) 를 실행한다. 여기서 POST는 컴퓨터에 연결된 각각의 하드웨어가 정상적으로 작동하는 지 확인하는 것이다. 만약 메모리가 정확하게 꽂혀있지 않으면 삡삡 하는 비프음(beep sound)가 난다고 한다.

이 펌웨어 들은 디스크 중에 어떤 장치가 부팅 장치인지 확인하는 작업을 수행한다. 요즘은 BIOS는 많이 보이지 않고 대부분 UEFI로 되어있다고 한다.

  • BIOS -> Master Boot Recorder ( 하드드시크 가장 앞쪽에 있는 섹터, 512 byte) , Primary Partition Table
  • UEFI -> GUID Partition Table

2-3. GRUB(GRand Unified Bootloader)

커널을 어떻게 메모리에 올릴건지(파라미터 값) 확인하고 운영체제를 로드한다.

여기서 신기한게 하나 있다.!!

커널을 메모리에 로드하려면 /boot 파일 시스템이 메모리에 마운트(mount) 되어야한다. /boot/vmlinuz에 커널을 메모리에 로드하고 커널에 필요한 파라미터 값이 들어 있기 때문이다. 하지만 파일시스템을 마운트하는 것은 운영체제가 한다… 즉 닭이 먼저냐 달걀이 먼저냐.. 가 되는 것이다. (나는 닭이 먼저라고 생각하다.)

이런 문제 때문에 우리가 진짜 사용하는 운영체제 말고 오르지 운영체제를 로드하기 위한 경령화된 운영체제가 존재한다. 이것이 바로 ramdisk이다. 이것음 초기 부팅시 커널에 필요한 하드웨어 드라이버와 파일 시스템도구를 제공하는 역할을 한다.

처음에 ramdisk의 파일시스템 /sysroot에 /(root) 파일 시스템을 마운트 한후에 부팅이 완료 되면 메모리에서 해제된다.

3. 운영체제가 메모리에 로드된다.

3-1. init / systemd (PID=1)

운영체제가 로드된 후 가장 처음으로 프로세스를 실행한다. 그 프로세스가 init/systemd를 실행하게 된다. init은 예전 버전으로 redhat 7 버전 이후로는 systemd 로 바뀌게 된다. 가장 큰 변화는 필요한 Target(논리적인 서비스 unit 묶음)을 병렬로 서비스를 시작한다는 점이다. 즉 init은 모든 서비스를 한줄로 세워서 부팅하는데 시간이 오래 걸렸다면 systemd는 이를 병렬로 처리할 수 있게 했다.

이제 모든 프로세스는 systemd를 부모로 하는 트리계층 구조로 생성되게 되는 것이다.

단말창에 psTree 라고 입력하면 각 프로세스의 계층구조를 알 수 있다. 우리는 여기서 제일 위에 systemd라는 process가 PID=1으로 실행 된것을 볼수 있다.

3-2. 터미널 실행

여기까지 부팅에 성공했다면 Target 값에 따라, 터미널을 띄워주거나 graphical.target이라면 로그인 GUI화면이 보이게 된다.
