CSAPP-1.컴퓨터 시스템으로의 여행

2025-02-28

1.1 정보는 비트와 컨텍스트로 이루어진다

컴퓨터 시스템 내부의 정보(디스크 파일, 메모리상의 프로그램, 데이터, 네트워크를 통해 전송되는 데이터)는 0 또는 1로 표시되는 비트들로 표시된다. 서로 다른 객체들을 구분하는 유일한 방법은 이를을 바라보는 컨텍스트 에 의해서 인데 동일한 일련의 바이트가 컨텍스트에 따라 다르게 해석될 수 있다.

특히 문자는 아스키(ASCII)표준을 따르는데 각각의 문자를 바이트(8bit) 길이의 정수값으로 나타낸다.

1.2 프로그램은 다른 프로그램에 의해 다른 형태로 번역된다 : 컴파일 시스템

프로그램은 ASCII 문자로 시작해서 컴파일러와 링커에 의해 바이너리 실행파일들로 번역되는 방식으로 다른 프로그램들에 의해 다른 형태로 번역된다.

  1. 전처리 단계
    전처리기 (cpp)는 #으로 시작하는 directive에 따라 #include<stdio.h> 헤더파일인 stdio.h를 프로그램 문장에 직접 삽입한다.

  2. 컴파일 단계
    hello.i 텍스트 파일을 어셈블리어로 번역하여 저장한다.

  3. 링크단계
    표준 C 라이블러리에 들어있는 printf를 호출하기 위해 printf.o을 결합해 hello.o 목적파일을 만든다. 이 파일은 실행가능 목적파일로 메모리에 적재되어 시스템에 의해 실행된다.

1.3 컴파일 시스템이 어떻게 동작하는지 이해하는 것은 중요하다.

  • 프로그램 성능을 최적화 하기위해
  • 링크 에러를 이해하기 위해
  • 보안 취약점을 피하기 위해

1.4 프로세서는 메모리에 저장된 인스트럭션을 읽고 해석한다.

컴퓨터 시스템은 응용프로그램을 실행하기 위해 함께 동작하는 하드웨어와 시스템 소프트웨어로 구성된다.

프로세서는 메인 메모리에 저장된 바이너리 인스트럭션을 읽고 해석한다. 이를 위해 컴퓨터는 대부분의 시간을 메모리, 입출력장치, CPU레지스터 간의 데이터를 복사하는데 쓰고 있다.

<하드웨어 구성>

  • 버스(Bus) : 컴포넌트 간에 바이트 정보를 전송하는 전기적 배선을 버스라고 한다. 버스는 일반적으로 워드 word라고 하는 고정크기의 바이트 단위로 데이터를 전송한다. (4바이트 또는 8바이트)

  • 입출력장치 : 시스템과 외부세계와의 연결을 담당한다.
    입출력장치는 I/O bus와 컨트롤러 또는 어뎁터를 통해 연결된다. 네트워크도 일종의 입출력장치라고 할수 있다.

    - 컨트롤러 (Controller) : 디바이스 자체가 칩셋이나 마더보드에 장착되는 형태
- 어뎁터 (Adapter) : 마더보드의 슬롯에 장착되는 카드

  • 메인메모리(RAM) : 메인 메모리는 프로세서가 프로그램을 실행하는 동안 데이터와 프로그램을 모두 저장하는 임시 저장소이다. 물리적으로 메인 메모리는 DRAM (Dynamic Random Access Memory) 칩들로 구성되어있다.

  • 프로세서: 주처리장치 (CPU)는 멤인 메모리에 저장된 인스트럭션들을 해독(실행)하는 엔진이다. 프로세서의 중심에는 워드 크기의 저장장치(=레지스터)인 프로그램 카운터가 있다. PC는 다음 실행할 메인 메모리의 인스트럭션 위치를 가리키고 있다.

인스트럭션의 요청에 따리 CPU는 다음과 같은 작업을 한다.

  • 적재(Load) : 메인메모리에서 레지스터로 한바이트 또는 워드 값을 가져온다.
  • 저장(Store) : 레지스터에서 메인메모리로 한 바이트 또는 워드 값을 저장한다.
  • 작업(Operate) : 두 레지스터 값을 ALU(수식/논리처리기) 로 가져와 수식연산을 수행한 후 저장한다.
  • 점프(Jump) : 다음 실행할 인스트럭션을 PC가 가르키도록 한다?

hello.c 가 실행되어 화면에 출력되려면

  1. 쉘프로그램에서 명령어를 읽어드린다.
  2. 실행파일을 디스크에서 메인 메모리로 로딩한다.
  3. 프로세서는 인스트럭션을 실행하기 시작한다.
  4. prinft를 하기 위해 “hello world” 스트링을 메모리로 부터 레지스터 파일로 복사한다.
  5. 디스플레이 장치로 전송하여 화면에 표시한다.

1.5 캐시가 중요하다.

1.6 저장장치들은 계층 구조를 이룬다.

데이터를 복사하는데 컴퓨터가 대부분의 시간을 쓰고 있는데 이를 더욱 빠르게 하기 위해 시스템의 저장 장치들은 계층구조를 가지고 있다. 그리고 각 계층 구조의 상부에 위치한 저장장치들은 하부의 장치들 보다 빠르고 비싸다. 상부의 저장장치들은 하부의 장치들을 위한 캐시 역할을 수행한다.

이런 역할이 가능한 이유는 프로그램이 특정 코드와 데이터를 자주 액세스하는 경향인 지역성이 있기 때문이다. 자주 액세스할 가능성이 높은 데이터를 캐시가 보관하도록 설정하면 빠른 캐시를 이용해서 대부분의 메모리 작업을 수행할 수 있게 된다.

1.7 운영체제는 하드웨어를 관리한다.

운영체제 커널은 응용프로그램과 하드웨어 사이에서 중간자의 역할을 수행한다. 응용프로그램이 하드웨어를 직접 잘못사용하는 것을 막기 위해 또 복잡하고 서로 다른 저수준의 하드웨어를 균일한 메커니즘을 제공해 응용프로그램이 사용할 수 있도록 하기 위해 운영체제는 다음과 같은 추상화를 제공한다. 따라서 응용프로그램은 복잡한 내부 동작을 고려하지 않으면서 HW를 사용할 수 있다.

  • 파일은 입출력장치의 추상화이다.
    모든 입출력장치는 시스템상에서 파일로 되어있다.
  • 가상메모리는 메인메모리와 디스크의 추상화이다.
    각 프로세스들이 메인 메모리를 독접하고 있는 것같은 환상을 제공한다. (프로그램 코드와 데이터, 힙, 공유라이브러리, 스택, 커널 가상메모리)
  • 프로세스는 프로세서, 메인메모리, 입출력장치의 추상화이다.
    운영체제는 시스템에서 한개의 프로그램만 실행되는 것 같은 착각에 빠지게 해준다.
  • 가상머신은 운영체제, 프로세서, 프로그램 모두를 포함하는 컴퓨터 전체의 추상화를 제공한다.

멀티코어 프로세서 와 멀티쓰레딩으로 프로세서가 한번에 더 많은 일을 할때(병렬성 Parallelism) 시스템 성능이 개선되었다.

멀티쓰레딩 (하이퍼쓰레딩)

하나의 CPU가 여러개의 제어 흐름을 실행할수 있게 해주는 기술이다. 이것은 프로그램 카운터나 레지스터파일 같은 여러개의 동일한 하드웨어를 가지고 있는 받면, 부동소수 연산기(ALU)와 같은 다른 부분은 한개의 하드웨어을 가지는 구조이다.
